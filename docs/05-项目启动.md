# 项目启动指南

## 环境要求

- Node.js >= 18
- Yarn (推荐) 或 npm

## 安装依赖

```bash
yarn install
# 或
npm install
```

## 开发环境

### 1. 配置环境变量

```bash
NEXT_PUBLIC_ENV=test
```

### 2. 启动开发服务器

```bash
# 使用测试环境配置
yarn dev:test

# 使用生产环境配置
yarn dev:prod

# 或手动指定
cp .env.test .env
yarn dev
```

访问 [http://localhost:3000](http://localhost:3000)

## 生产构建

```bash
# 测试环境构建
yarn build:test

# 生产环境构建
yarn build:prod
```

构建产物输出到 `out/` 目录（静态导出）

## 可用脚本

| 命令 | 说明 |
|------|------|
| `yarn dev:test` | 启动测试环境开发服务器 |
| `yarn dev:prod` | 启动生产环境开发服务器 |
| `yarn build:test` | 构建测试环境 |
| `yarn build:prod` | 构建生产环境 |
| `yarn lint` | 运行 ESLint |

## 配置说明

### 核心配置文件

```typescript
// types/constant.ts
export const WEBSITE_CONFIG = {
  // 网站信息
  appTitle: "Next App",
  appName: "Next App",
  
  // 链配置
  supperChains: [sepolia],
  defaultChain: sepolia,
  
  // WalletConnect 配置
  walletConnectProjectId: "YOUR_PROJECT_ID",
  
  // API 地址
  API_BASE_URL: "https://api.example.com/",
  
  // 功能开关
  needSignLogin: true, // 是否需要签名登录
  
  // 主题/语言
  defaultLanguage: LANGUAGE_ENUM.en,
  defaultTheme: THEME_ENUM.light
}
```

## 常见问题

### 1. 钱包连接失败

检查 `WEBSITE_CONFIG.walletConnectProjectId` 是否配置正确  
前往 [WalletConnect Cloud](https://cloud.walletconnect.com/) 创建项目

### 2. API 请求 401

- 检查 `API_BASE_URL` 是否正确
- 确认 `needSignLogin` 配置
- 清除 localStorage 重新登录

### 3. 构建报错

```bash
# 清除缓存重新安装
rm -rf node_modules yarn.lock
yarn install
yarn build:prod
```

## 部署

项目支持静态导出，构建后的 `out/` 目录可直接部署到：

- Vercel
- Netlify
- GitHub Pages
- 任何静态服务器

```bash
# 构建
yarn build:prod

# 部署 out/ 目录
```

## 开发建议

1. **首次开发**：先查看 `app/example/page.tsx` 了解组件使用
2. **Web3 调试**：使用 Sepolia 测试网，[获取测试币](https://sepoliafaucet.com/)
3. **样式开发**：使用 Styled-Components，支持主题切换
4. **状态管理**：优先使用 Redux，钱包状态已内置
